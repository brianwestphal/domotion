# Domotion docs

Requirements / design docs for the rendering pipeline. Numbered to match the order they were authored; numbering does not imply dependency order.

> **Cross-platform note**: Domotion is an npm package and is expected to function on macOS, Linux, and Windows. Today the rendering pipeline is fully calibrated only against Chromium-on-macOS (CoreText-based fallback chain); Linux (fontconfig — Noto / DejaVu / Liberation) and Windows (DirectWrite — Arial / Consolas / Segoe UI Symbol / Cambria Math) calibration is roadmap work tracked in DM-258 (path discovery) → DM-259 (Linux) / DM-260 (Windows) / DM-261 (bundled fallback fonts) → DM-262 (per-platform CI). Treat any macOS-only code path as debt to flag, not a design choice — match Chromium's actual painted output on each target platform.

| [26-self-contained-svgs.md](26-self-contained-svgs.md) | `embedRemoteImages(tree)` pre-pass that fetches every http(s) image URL referenced by the captured tree and inlines the bytes as `data:` URIs. Opt-in via `selfContained: true` in `CaptureOptions`; demos run it unconditionally. |
| [27-image-resize-on-embed.md](27-image-resize-on-embed.md) | `resizeEmbeddedImages` pre-pass that downscales each inlined image to its render rect × `hiDPIFactor` (default 2×) and re-encodes as PNG. Targets 50–80 % SVG size reduction on news-site captures with no visible diff at the captured viewport. Opt-in via `embedRemoteImagesResize: true`; only runs when `selfContained: true` has populated the data-URI cache. |
| [28-conic-gradient.md](28-conic-gradient.md) | `conic-gradient` / `repeating-conic-gradient` background layers via a pattern-raster pre-pass (sharp → PNG → `<pattern><image>`). Covers `from <angle>`, `at <position>`, hard stops, multi-layer composition. Animation deferred. |
| [29-dark-mode.md](29-dark-mode.md) | Dark-mode rendering: caller-chooses `colorScheme` plumbed through `captureElementTree`, `color-scheme="dark"` attr on the emitted SVG, empirically calibrated dark form-control palette in `src/form-controls.ts`, and capture-side `rootBgComputed` so the transparent-root fallback honors Chromium's resolved value. |
| [25-keyframe-extraction-for-symbols.md](25-keyframe-extraction-for-symbols.md) | t=0 paint-state baking for CSS-animated `<symbol>` contents — `getComputedStyle().transform` (composed with `transform-origin`) and presentation attrs are baked onto the inlined subtree at capture time. Replaces DM-499's animated-subtree raster fallback. The animation itself doesn't survive; the captured frame matches Chrome's paint. |
| [24-inline-svg-icon-references.md](24-inline-svg-icon-references.md) | Inline SVG icons that use `<use href="#id">` to reference symbols defined in a separate hidden `<defs>` SVG (apple.com country dropdown checkmark, social icons, etc.) — resolve-and-inline at capture time, rasterise fallback when refs can't be resolved. |
| [22-mask-element-paint-references.md](22-mask-element-paint-references.md) | CSS `mask-image: element(#id)` paint references — per-id `page.screenshot` of the referenced element, emitted as `<image>` inside `<mask>` with `mask-type="luminance"` for match-source. Same-document only; deduped per id. |
| [18-background-clip-text.md](18-background-clip-text.md) | CSS `background-clip: text` (gradient/image-fill inside glyph shapes) — bg layer's def is reused as the source of an SVG `<mask>` over a colored `<rect>` covering the headline. Why mask and not `<clipPath>` (Chromium doesn't honor `<use>` inside `<clipPath>`). |
| [23-css-sprite-icons.md](23-css-sprite-icons.md) | CSS sprite icons (image-replacement idiom: `text-indent: -9999px; overflow: hidden; background: url(/sprite.png) -X -Y no-repeat`) — replaced-element raster path captures the painted icon rect, with off-screen text suppressed and `aria-label` carried through as SVG `<title>`. |

The docs preserve their historical SK-XXXX references — those are the slicekit Hot Sheet tickets that originally drove each feature. Treat the ticket numbers as opaque pointers to design context, not as live work tracking.
