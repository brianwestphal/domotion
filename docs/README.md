# Domotion docs

Requirements / design docs for the rendering pipeline. Numbered to match the order they were authored; numbering does not imply dependency order.

> **Cross-platform note**: Domotion is an npm package and is expected to function on macOS, Linux, and Windows. Today the rendering pipeline is fully calibrated only against Chromium-on-macOS (CoreText-based fallback chain); Linux (fontconfig — Noto / DejaVu / Liberation) and Windows (DirectWrite — Arial / Consolas / Segoe UI Symbol / Cambria Math) calibration is roadmap work tracked in DM-258 (path discovery) → DM-259 (Linux) / DM-260 (Windows) / DM-261 (bundled fallback fonts) → DM-262 (per-platform CI). Treat any macOS-only code path as debt to flag, not a design choice — match Chromium's actual painted output on each target platform.

Opt-in via `embedRemoteImagesResize: true`; only runs when `selfContained: true` has populated the data-URI cache. |

| [25-keyframe-extraction-for-symbols.md](25-keyframe-extraction-for-symbols.md) | t=0 paint-state baking for CSS-animated `<symbol>` contents — `getComputedStyle().transform` (composed with `transform-origin`) and presentation attrs are baked onto the inlined subtree at capture time. Same-document only; deduped per id. |
| [18-background-clip-text.md](18-background-clip-text.md) | CSS `background-clip: text` (gradient/image-fill inside glyph shapes) — bg layer's def is reused as the source of an SVG `<mask>` over a colored `<rect>` covering the headline. Why mask and not `<clipPath>` (Chromium doesn't honor `<use>` inside `<clipPath>`). |

| [33-element-out-of-viewbox-hiding.md](33-element-out-of-viewbox-hiding.md) | Phase 2 of DM-599 animation performance work — element-level `display: none` for content outside the viewBox. Composition-time pass driven by `CapturedElement.{x,y,width,height}` + the intra-frame animation's explicit `from`/`to`. |
| [32-svg-optimization.md](32-svg-optimization.md) | SVG size reduction — `optimizeSvg(svg)` (SVGO post-pass, plugin set tuned for path-mode text) and `gzipSvg(svg)` (zlib gzip for `.svgz` output). CLI auto-detects `.svgz` from `-o` and implies `--optimize` unless `--no-optimize` is passed. |
| [31-region-feedback.md](31-region-feedback.md) | Region-scoped feedback in the demos-review tool — user drags rectangles on the `expected`/`actual`/`diff` triplet shown by `npm run demos:review`; rectangles are serialized into a `REGIONS:` block in the Hot Sheet ticket comment; the AI iteration loop produces one crop per rectangle (under `tests/output/region-crops/DM-{id}/{noteId}/`) and feeds them as primary visual evidence alongside the full attachments. Contract for DM-570; implementation tickets filed separately. |
| [37-scroll-pattern-grammar.md](37-scroll-pattern-grammar.md) | Canonical EBNF + semantics reference for the scroll-pattern language: top-level sequencing, parenthesised groups, `until` clauses, direction/axis resolution, speed-vs-duration semantics (and the CLI flags / config keys / programmatic options that drive `defaultSpeed`), easing, and worked examples. The parser (`src/scroll/pattern.ts`) and executor (`src/scroll/executor.ts`) both point here. |
| [46-cli-npx-invocation.md](46-cli-npx-invocation.md) | Contract for running the CLI as an npm bin (`npx domotion-svg …`): the `bin` mapping + shebang, build-before-publish, runtime version read from `package.json` (no hardcoded literal), Node 22 + Playwright-Chromium prerequisites (auto-installed on first run), and the unsupported git-URL-invocation caveat. |
