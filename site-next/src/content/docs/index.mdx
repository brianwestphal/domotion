## One file. It moves. It scales. It embeds anywhere.

A Domotion output is a single `<svg>` — no external fonts, images, or scripts.
Drop it into a README, an npm page, an email, a slide, or a docs site and it
plays its CSS animation, lazy-loads, stays crisp at any zoom, and carries real
selectable text. No video file, no iframe, no re-encode per device.

## A few things it does

<CardGrid>
  <Card title="Faithful capture" icon="laptop">
    Renders real HTML/CSS in headless Chromium and emits the same pixels as
    native SVG — text becomes glyph `<path>`s, so it looks identical in every
    browser.
  </Card>
  <Card title="Terminal sessions" icon="seti:shell">
    Turn an asciinema recording into an animated terminal — real text, real ANSI
    color, native SVG (no raster frames).
  </Card>
  <Card title="Templates" icon="puzzle">
    A few flags become a polished animated SVG — lower-thirds, charts, kinetic
    text, device mockups, and more.
  </Card>
  <Card title="Compositing" icon="document">
    Nest animated layers inside animated layers — a terminal window resizing on
    a desktop, a scrolling site inside a browser bezel.
  </Card>
</CardGrid>

## A whole application — captured as one SVG

Not a snippet: a complete, stateful product. This is a real Domotion capture of
**Glassbox**, an AI code-review tool, walking its entire flow — launch, AI risk
triage, a split diff, a line annotation, completing the review, exporting the
feedback, and an agent applying the fix — in one infinitely-looping, self-contained SVG.
